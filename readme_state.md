# Interactive Equipment Catalog - Текущее состояние (Версия 1.0)

## Обзор архитектуры
Проект представляет собой интерактивный 3D-каталог оборудования, написанный полностью на "ванильных" HTML, CSS и JavaScript с использованием библиотеки **Three.js** (избегая тяжелых фреймворков вроде React/R3F для максимальной производительности в браузере).

## Основной функционал

### 1. Режим Сетки (Grid View)
- Интеллектуальная сетка, растянутая на 90% ширины экрана, с 4 равными колонками.
- Для обхода жесткого лимита браузеров на количество WebGL контекстов (макс 16 штук) реализован **механизм Offscreen Canvas rendering**. Каждая 3D модель рендерится в скрытом буфере, после чего ее "фотография" (pixels) копируется на обычный 2D canvas в ячейке. После этого WebGL контекст уничтожается.
- Модели масштабируются и центрируются с помощью математических просчетов диагонали Bounding Box (`fitCameraFrontal` и `fitCameraFloorLevel`), что исключает обрезание моделей по краям холста.
- Уникальные правила позиционирования: плоские/маленькие модели (XBOX, Platforma RGB) приподнимаются, чтобы парить в центре.

### 2. Режим Карусели (Carousel View)
- Активируется по клику на любую модель в сетке.
- Отображает текущую модель по центру (огромного размера и с интерактивным управлением) и 4 соседние модели на заднем фоне (по 2 с каждой стороны с эффектом затухания и отдаления).
- Реализована **Drag-to-Rotate / Swipe механика** (мышь и Touch-события на мобилках).
- Угол вращения моделей ограничен (`clamp` на ±30-60 градусов) с физикой возвращения в центр при отпускании курсора (`inertia`).

### 3. Загрузчик и Материалы (GLTFLoader)
- Все модели загружаются один раз и кэшируются (`modelCache`).
- Написана кастомная логика подмены материалов стекол на лету. При совпадении имен материалов (`glass`, `стекло`, `clean glass` для аркадного автомата) материал получает прозрачность (`opacity: 0.2`), блики стекол (`roughness: 0.05`, `metalness: 0.5`) и правильный порядок рендеринга (`renderOrder = 1`).

### 4. Освещение (Текущее)
В текущей версии реализован базовый Three.js свет:
- `AmbientLight` (белый свет, интенсивность 0.8) для базовой видимости теней.
- `DirectionalLight 1` спереди/сбоку (интенсивность 1.5).
- `DirectionalLight 2` сзади/заполняющий (интенсивность 0.4).
